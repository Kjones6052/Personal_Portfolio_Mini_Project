Contact Manager Application

Link to Project Repository

Project Description

The contact manager application was created and designed to organize a group of contacts into a dictionary where the full name is the key and the values are the person's contact information. The application uses a command line interface to allow a user to add a contact, search for and display the contact details of a contact, update a contact, display all contacts, delete a contact, and importing and exporting the contacts to a text file. The application utilizes regular expressions to verify data is in the correct format, and a modular design structure that seeparates the code into files that are dedicated to a single purpose(ie; "add a contact" is in its own file) with a main file called "contact_manager" that functions as the main script and user interface.

Library Management System

Link to Project Repository

Project Description

The library management system is an application which was created and designed to do what it sounds like, manage a system of books, authors, and users in a library. Utilizing class structures to create objects the application allows a user to go through a menu system with end results allowing the user to complete a series of tasks. The tasks the applications completes are divided into groups for book operations, author operations, and user operations. For books the user can add a new book, borrow a book, return a book, view a list of all books, and search for and view details of a book. For users we can add a new user, search for and view user details including a list of borrowed books, and display a list of all users. For authors the user is able to add a new author, search for and view details of an author, and view all authors. The application also uses regular expressions to verify data input is received in the correct format.

E Commerce API

Link to Project Repository

Project Description

This project is a great mix of all the tools and abilities that I learned at Coding Temple. The first step in creating this project was to create a Python Virtual Environment where I then installed several packages: MySQl Connector, Flask, Marshmallow, and SQLAlchemy. From these packages I imported the following tools: jsonify, request, fields, SQLAlchemy Error, and ValidationError. Schemas were then created for validation of data format and types, and Models were created to format the tables used in the database. I used Postman to text the input and output of the program verifying that all operations were successful or unsuccessful according to input. This project was probably the most fun to create in my codingjourney so far, because it allowed me to utilize almost all of the tools and skills I obtained during my education with Coding Temple.
